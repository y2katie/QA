Testing Checklist

  • Identify all interactive elements with visible text labels
  • Check accessible name using DevTools or screen reader
  • Verify visible label text is included in accessible name
  • Test with speech input software if available

Testing Checklist

  • Test all form controls for unexpected behavior on change
  • Verify dropdowns don't auto-submit or navigate
  • Check that context changes are user-initiated

SC 2.5.2: Pointer Cancellation

Functions triggered on pointer down-event must be cancellable, reversible, or essential.

Testing Checklist

  • Test mouse down vs mouse up behavior
  • Verify drag-away cancellation works
  • Check for undo mechanisms on immediate actions
  • Identify essential functions (drawing, games)

SC 4.1.2: Name, Role, Value (Advanced)

Custom components must expose proper name, role, and value to assistive technologies.

‚úÖ Proper Custom Toggle

Role: switch | State: not pressed

Required ARIA Patterns

  • Modal: role="dialog", aria-labelledby, aria-modal
  • Combobox: role="combobox", aria-expanded, aria-controls
  • Slider: role="slider", aria-valuemin/max/now
  • Toggle: role="switch", aria-checked

Testing Checklist

  • Inspect accessibility tree in DevTools
  • Verify role, name, and state are exposed
  • Test with screen reader navigation
  • Check keyboard interaction patterns
