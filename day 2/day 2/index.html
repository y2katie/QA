Page Meaning & Structure for QA

Why Page Meaning Matters

Building on Day 1's screen reader knowledge, page meaning ensures content is logically structured and semantically rich for all users.

QA Impact of Poor Page Structure

  • Screen readers can't navigate efficiently (from Day 1)
  • SEO suffers from unclear content hierarchy
  • Users with cognitive disabilities get confused
  • Automated testing tools fail to understand content

Semantic HTML Elements

Semantic vs Generic Elements

✅ Semantic HTML

<article>
  <header>
    <h2>Article Title</h2>
    <time datetime="2024-01-15">Jan 15, 2024</time>
  </header>
  <p>Article content...</p>
  <footer>
    <p>By <cite>Author Name</cite></p>
  </footer>
</article>

Clear meaning and structure for all users

❌ Generic HTML

<div class="article">
  <div class="header">
    <div class="title">Article Title</div>
    <div class="date">Jan 15, 2024</div>
  </div>
  <div class="content">Article content...</div>
  <div class="footer">
    <div>By Author Name</div>
  </div>
</div>

No semantic meaning, relies only on visual styling

Page Metadata & Document Information

Essential Metadata for Accessibility

Page Title

<title>Product Name - Category - Site Name</title>

Descriptive, unique, front-loaded with key info

Language Declaration

<html lang="en">
<span lang="es">Hola mundo</span>

Helps screen readers pronounce content correctly

Skip Links

<a href="#main" class="skip-link">
  Skip to main content
</a>

Allows keyboard users to bypass navigation
